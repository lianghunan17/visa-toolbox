签证工具箱上线前最终验收

这是当前项目进入正式公开前的最后验收页，适合在真正发布前快速过一遍。

一，必须为绿

  • npm run preflight 已通过
  • /health 返回 ok: true
  • /ready 返回 ok: true
  • 后台诊断面板全绿
  • 授权创建、激活、验权至少走通一遍

二，正式环境不能犯的错

  • 不要把前端继续指向 127.0.0.1
  • 不要把 service role key 放进前端
  • 不要恢复 NODE_TLS_REJECT_UNAUTHORIZED=0
  • 不要让 admin.local.html 当正式后台入口

三，推荐阅读顺序

  1. GO_LIVE_RUNBOOK.md
  2. PUBLIC_LAUNCH_CHECKLIST.md
  3. PRODUCTION_API_INJECTION.md
  4. FRONTEND_DEPLOYMENT.md
  5. ADMIN_DEPLOYMENT.md

四，正式公开顺序

  1. 先部署正式 API
  2. 确认正式 /health 和 /ready 变绿
  3. 再部署正式前端
  4. 显式注入正式 API 域名
  5. 先灰度给 3 到 10 个真实用户
  6. 灰度稳定后再扩大公开

五，当前本地成果入口

  • 运行态展示页
  • 后台本地入口
  • 本地 /health
  • 本地 /ready
